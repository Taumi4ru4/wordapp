# 単語帳アプリ（PWA + Firebase版）

家のPCで単語を追加 → 通勤電車（オフライン含む）でスマホで復習、を実現するアプリです。

- スマホのホーム画面に「アプリ」として追加できます（PWA）
- 電波がない満員電車でも動作します（一度開いたことがあれば）
- Wi-Fiにつながった時に自動でクラウド(Firebase)と同期されます
- データは自分専用（ログインしたあなたしかアクセスできません）

---

## ステップ1: Firebaseプロジェクトを作る（無料）

1. https://console.firebase.google.com/ にアクセスし、Googleアカウントでログイン
2. 「プロジェクトを作成」→ 好きな名前（例: `tangocho-app`）を入力 → 作成
   - Googleアナリティクスは「無効」でOKです
3. 左メニューの「構築」→「Firestore Database」→「データベースの作成」
   - ロケーションは `asia-northeast1`（東京）などお好きな場所を選択
   - 「本番環境モード」で開始（後でルールを設定します）
4. 左メニューの「構築」→「Authentication」→「始める」
   - 「Sign-in method」タブ →「メール / パスワード」を有効化

## ステップ2: ウェブアプリを登録し、設定値を取得する

1. プロジェクトのトップページ（⚙アイコン →「プロジェクトの設定」）
2. 「マイアプリ」→ `</>`（ウェブ）のアイコンをクリック
3. アプリ名を入力（例: `tangocho-web`）→ 登録
4. 表示される `firebaseConfig` の値（apiKey, authDomain, projectIdなど）をコピー
5. このフォルダの **`firebase-config.js`** を開き、コピーした値で書き換えて保存

```js
const firebaseConfig = {
  apiKey: "AIzaSy....",
  authDomain: "tangocho-app.firebaseapp.com",
  projectId: "tangocho-app",
  storageBucket: "tangocho-app.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef"
};
```

## ステップ3: セキュリティルールを設定する（自分のデータを守る）

1. Firebaseコンソール →「Firestore Database」→「ルール」タブ
2. このフォルダの `firestore.rules` の内容を貼り付けて「公開」をクリック

これで、ログインした本人以外は自分のデータを読み書きできなくなります。

## ステップ4: デプロイする（GitHub Pagesがおすすめ・無料）

1. GitHubで新しいリポジトリを作成（Private/Publicどちらでも構いません。中身はコードのみで、単語データはFirestore側にあるため、リポジトリをPublicにしても単語の内容は公開されません）
2. このフォルダ内の以下のファイルをすべてアップロード:
   - `index.html`
   - `app.js`
   - `firebase-config.js`（自分の値に書き換えたもの）
   - `manifest.json`
   - `service-worker.js`
   - `icon-192.png`
   - `icon-512.png`
3. リポジトリの「Settings」→「Pages」→ Source を「Deploy from a branch」→ ブランチを`main`、フォルダを `/(root)` に設定 → Save
4. 数分後、`https://(あなたのユーザー名).github.io/(リポジトリ名)/` というURLでアプリにアクセスできます

### Firebase独自のドメインを使う場合（任意・より簡単）
GitHubを使わず、Firebase Hosting（無料）でも公開できます。Node.jsが使える環境なら:
```
npm install -g firebase-tools
firebase login
firebase init hosting
firebase deploy
```

## ステップ5: スマホで使う

1. スマホのブラウザ（Safari/Chrome）で、デプロイ後のURLを開く
2. ログイン画面でメールアドレス・パスワードを登録（最初の1回だけネット接続が必要）
3. Safariなら共有ボタン→「ホーム画面に追加」、Chrome(Android)なら「ホーム画面に追加」を選択
4. これでアプリのアイコンができ、オフラインでも開けるようになります

## ステップ6: 以前のデータを移行する

1. PCのブラウザで同じURLを開き、ログイン
2. ホーム画面の「📥 以前のデータをインポート」を開く
3. このフォルダ内の `tangocho_data_export.json`（以前のPythonコードのデータをそのまま変換したものです）を選択
4. これで163個の単語データがそのまま引き継がれます

---

## 日常の使い方

- **家のPC**: ログインして「➕新しい単語を追加する」から単語を追加 → 自動でクラウドに保存されます
- **通勤電車**: スマホでアプリを開く（一度家のWi-Fiで開いておけば次回からオフラインでもOK）→「今日の学習を始める」→ 復習 → 間違えた単語はその場で再挑戦
- **データの流れ**: Wi-Fiにつながるたびに、PCとスマホのデータが自動的にクラウド経由で同期されます

## 仕組み・ロジックについて

元のPythonプログラムと同じ間隔反復のロジックを使っています:
- 正解するとステージが1つ上がり、次に出題されるまでの日数が伸びます（0,0,1,4,19,29日後）
- ステージ5で正解すると「習得済み」リストに移動します
- 間違えるとステージ0に戻り、その日のうちに再度出題されます
- 1日ごとに新しい単語を `tuika` 個（設定で変更可）、新規プールからランダムに追加します

## トラブルシューティング

- **ログインできない/「通信エラー」と出る**: 初回ログインだけはネット接続が必要です。Wi-Fiに繋いでから試してください。
- **オフラインで開いたらまっさら（ログイン画面）になる**: 一度オンラインの状態でログインしてから、二回目以降オフラインで開く必要があります。
- **スマホとPCでデータが一致しない**: 両方をオンラインの状態で一度開くと自動的に同期されます（リアルタイムではなく、アプリを開いたタイミングで同期される設計です）。
